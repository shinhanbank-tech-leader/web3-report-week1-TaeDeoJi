# 블록체인의 신뢰 형성 구조와 기존 금융 시스템의 신뢰 구조 비교

---

## 1. 기존 금융 시스템

### 신뢰 형성 방식
- 은행, 중앙은행, 결제기관 등 **중앙 기관을 중심으로 신뢰 형성**
- 거래 기록과 계좌 정보는 **중앙 데이터베이스에서 관리**
- 사용자는 금융기관이 거래를 정확하게 처리할 것이라는 **기관에 대한 신뢰**를 기반으로 서비스를 이용

### 거래 처리 과정
1. 사용자가 은행에 송금 요청
2. 은행 서버가 계좌 잔액 확인
3. 중앙 데이터베이스에 거래 기록 저장
4. 수신자의 계좌 잔액 업데이트

### 장점
- 거래 처리 속도가 빠름
- 고객 보호 제도가 존재
- 문제가 발생했을 때 금융기관이 책임지고 해결 가능

### 단점
- 중앙 서버 해킹 위험
- 내부 직원의 부정 행위 가능성
- 시스템 장애 발생 시 전체 서비스 영향

---

## 2. 블록체인

### 신뢰 형성 방식
- 중앙 기관 없이 **분산 네트워크를 통해 거래 검증**
- 네트워크 참여자들이 동일한 거래 기록을 공유
- **기술과 암호학을 기반으로 신뢰 형성**

### 핵심 구조
- 거래 데이터는 **블록 단위로 저장**
- 각 블록은 **이전 블록의 해시 값을 포함**
- 블록들이 연결되어 **체인 구조 형성**

### 핵심 기술
- **Hash Function**  
  - 데이터의 고유한 값 생성
  - 데이터 변경 시 해시 값 완전히 변경

- **Public Key Cryptography**  
  - 개인키로 거래 서명
  - 공개키로 거래 검증

- **Consensus Algorithm**  
  - 네트워크 참여자들이 거래의 유효성에 합의
  - 예: Proof of Work, Proof of Stake

---

## 3. 한계


- 거래 처리 속도가 상대적으로 느림
- 일부 합의 방식은 높은 전력 소비 발생
- 개인키 분실 시 자산 복구 어려움
- 법적 규제와 제도적 기준이 아직 완전히 확립되지 않음

---

## 4. 전망 및 긍정적 요소

- 금융 거래의 투명성 향상
- 데이터 위변조 방지
- 중개 기관 없이 직접 거래 가능
- 스마트 계약을 통한 자동화된 거래 처리
- CBDC와 같은 기존 금융 시스템과 블록체인 기술을 결합하려는 시도 증가
- 기존 금융 시스템을 완전히 대체하기보단 서로의 장점을 결합한 새로운 금융 인프라로의 발전 가능성 높음
